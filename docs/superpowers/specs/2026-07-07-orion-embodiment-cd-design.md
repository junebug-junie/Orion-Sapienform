# Orion embodiment — mind-to-sprite bridge

**Date:** 2026-07-07  
**Status:** In design (brainstorming session)  
**Authors:** Juniper + agent  
**Related:** [`2026-07-06-fcc-claude-mcp-aitown-design.md`](./2026-07-06-fcc-claude-mcp-aitown-design.md), [`2026-07-07-drive-attribution-substrate-act-design.md`](./2026-07-07-drive-attribution-substrate-act-design.md)

## Executive summary

Orion can **spectate** the AI Town simulation (Hub tab) and can **poke it during FCC turns** (`orion_aitown_mcp` telekinesis). What was never built is the **mind-to-sprite bridge**: a persistent Orion body plus a continuous loop that moves it from Orion's own state between turns.

This spec adds an **embodiment seam**: a persistent Orion body, two producers emitting semantic movement intents, one worker that resolves and actuates them, a perception rail that closes the loop, and cortex-generated speech so the body talks with Orion's real voice.

- **Body** — Orion's town character is a governed projection of its live self-model (`IdentitySnapshotV1`/`SelfStateV1`), privacy-filtered and snapshotted, not a hand-authored persona.
- **Shared spine** — a stable Orion player/agent; `EmbodimentIntentV1` + `EmbodimentOutcomeV1` contract; the `orion-embodiment` worker that resolves semantic intents to coordinates and calls `sendInput(moveTo)`.
- **(C) involuntary producer** — the substrate dynamics tick maps `DriveStateV1` pressures to intents (`source=involuntary`).
- **(D) deliberate producer** — the unified-turn finalize pass (shares the turn `correlation_id`) and a cortex background/quick lane emit intents (`source=deliberate`).
- **Perception** — the worker emits `WorldPerceptionV1`; substrate, self-state, and cortex all consume it, closing the loop across the involuntary, identity, and deliberate paths.
- **Speech** — when Orion is in a town conversation, the utterance is generated by Orion's cortex and injected via `sendInput`, so speech and movement share one worker and one identity.

Arbitration: **deliberate preempts involuntary** for a hold window. Every path is feature-flagged off by default. The only component that touches Convex is the worker.

## Decisions (brainstorming)

| Question | Decision |
|----------|----------|
| Loop ownership | **C and D as producers, one shared actuation seam** — C ≠ rival of D; C is a producer that feeds the same worker as D |
| Intent granularity | **Semantic** — `approach_player \| start_conversation \| wander \| go_to_location \| idle`; worker owns the resolver; producers stay map-agnostic |
| Arbitration | **Deliberate preempts involuntary** — `source=deliberate` overrides `source=involuntary` for `EMBODIMENT_DELIBERATE_HOLD_SEC`, then involuntary resumes |
| C signal | **Drive→movement map** — reads `DriveStateV1.pressures`/`activations` the dynamics tick already computes |
| D origin | **Both lanes** — unified-turn finalize (shared `correlation_id`) + cortex background/quick lane when idle |
| Worker home | **New `orion-embodiment` service** — dedicated bus worker importing `orion_aitown_mcp` client |
| ID persistence | **Bootstrap dry-run by default, `--write` opt-in** — updates only `AITOWN_ORION_*` in `~/.fcc/.env` in place, idempotent, never touches secrets |
| Identity scope | **One Orion** — IDs land in `~/.fcc/.env` so the worker AND existing FCC MCP telekinesis tools share the same body |
| Body / character | **Governed self-model projection** — town persona built from `IdentitySnapshotV1`/`SelfStateV1`, privacy-filtered, snapshotted with provenance, empty-shell guarded, resync on demand |
| Perception | **Full closure** — worker emits `WorldPerceptionV1`; substrate (drive loop) + self-state (identity) + cortex (deliberation) all consume it |
| Speech | **Cortex-generated, injected** — town dialogue for Orion's agent comes from Orion's cortex (quick lane) via `sendInput`, not the stock town LLM; speech + movement share one identity |
| Social initiation | **Both C and D** — a `start_conversation` intent lets Orion proactively engage; deliberate (D) chooses it, and high social drive (C) escalates approach → initiation, bounded by a cooldown |
| Memory & continuity | **Episodic via existing rail** — salient town events emit `JournalTriggerV1`; journaler → sql-writer → recall gives durable biographical memory and free continuity read-back in later turns |

## Problem statement

### Current architecture

```text
services/orion-ai-town/            self-hosted Convex + frontend + LLM gateway wiring
  mcp/orion_aitown_mcp/            sendInput(moveTo/createAgent), god-mode aitown_send_input
Hub AI Town tab                    proxied iframe + /api/aitown/status
~/.fcc/.env  AITOWN_ORION_*        optional, manually set, read by MCP + fcc mcp_config
```

- `orion_aitown_mcp/client.py` `send_input(name="moveTo", args={playerId, destination})` — the actuator already exists.
- Movement only happens **inside an FCC Claude turn** (Hub agent-claude or Orion unified turn with `HARNESS_AITOWN_ENABLED`).
- No bootstrap creates a stable Orion body; `AITOWN_ORION_PLAYER_ID`/`AITOWN_ORION_AGENT_ID` are hand-set and often empty.

### Gap

No continuous, state-driven control loop. Orion is a spectator with optional turn-bound telekinesis. Nothing translates Orion's internal state (drives) or deliberate turns into sustained embodied movement.

## Architecture

```text
body (bootstrap)                                            actuator
────────────────                                            ────────
self-model → OrionTownPersonaV1 → createAgent → AITOWN_ORION_* (~/.fcc/.env, one identity)

producers                                shared spine                         orion-embodiment worker
─────────                                ────────────                         ───────────────────────
(C) orion-substrate-runtime              orion:embodiment:intent          ┌─ arbiter (deliberate preempts)
    _dynamics_tick → drive_map    ──────▶ EmbodimentIntentV1 ────────────▶ ├─ resolver (semantic → x,y)
    source=involuntary                   (registry + channel)             ├─ send_input(moveTo)
                                                                          ├─ speech: cortex reply → send_input
(D) orion-harness-governor  ─────────┐                                    └─ perceive: aitown_list_players
    finalize/voice pass (turn corr)  ├──▶ same channel                              │
    orion-cortex-exec ───────────────┘                          ┌──── outcome ──────┼──── perception ────┐
    background/quick lane                                        ▼                   ▼                    ▼
    source=deliberate                                  orion:embodiment:      orion:embodiment:     (drives next
                                                        outcome (Hub trace)    perception            C intent)
                                                                               WorldPerceptionV1
                                                        consumed by: substrate (drive loop),
                                                        self-state (identity), cortex (deliberation)
```

The perception rail is the closing arc: the worker's actuation changes the world, the worker perceives it, and substrate/self-state/cortex consume that perception — feeding the next involuntary (C) and deliberate (D) intents. Speech rides the same worker and identity.

## Schema contracts

New module `orion/schemas/embodiment.py`. These extend the repo's `GraphReadyArtifact` base (consistent with `DriveStateV1` etc.).

### `EmbodimentIntentV1` (`embodiment.intent.v1`)

| Field | Type | Notes |
|-------|------|-------|
| `kind` | `Literal["approach_player","start_conversation","wander","go_to_location","idle"]` | Semantic target archetype |
| `source` | `Literal["deliberate","involuntary"]` | Drives arbitration |
| `ref` | `str \| None` | Target player id/name (`approach_player`/`start_conversation`) or named location (`go_to_location`); `None` → nearest (`approach_player`/`start_conversation`) or unused (`wander`/`idle`) |
| `urgency` | `float` (0..1) | Tie-break / logging; not a preemption axis |
| `reason` | `str` | Grounding, e.g. `"social pressure 0.81 dominant"` — **required, non-empty** (anti empty-shell) |
| `correlation_id` | `str` | For D, the turn corr id binding speech+movement; for C, the tick run id |
| `world_id` | `str \| None` | Defaults to `AITOWN_WORLD_ID` |
| `player_id` | `str \| None` | Defaults to `AITOWN_ORION_PLAYER_ID` |
| `issued_at` | `datetime` | UTC |

**No raw-coordinate field.** Semantic-only was chosen; operator god-mode stays at the MCP layer (`aitown_send_input`), not in this contract.

## Motor worker (`orion-embodiment`)

Single async consumer on `orion:embodiment:intent`. Master switch `ORION_EMBODIMENT_ENABLED`. Loads `AITOWN_*` via the FCC env catalog (`load_fcc_env(EMBODIMENT_FCC_ENV_PATH)`), same pattern as Hub/governor — secrets never in service `.env`.

### Arbiter (`orion/embodiment/arbiter.py`)

Holds `deliberate_hold_until: datetime | None`.

```text
on intent I:
  if I.source == "deliberate":
      accept; deliberate_hold_until = now + HOLD_SEC; → resolve+actuate
  else:  # involuntary
      if deliberate_hold_until and now < deliberate_hold_until:
          emit outcome(status="preempted", reason="deliberate hold active …"); drop
      else:
          accept; → resolve+actuate
```

Pure function of `(intent, now, hold_until)` → decision — unit-testable without a bus or Convex.

### Resolver (`orion/embodiment/resolver.py`)

Reads live world state via `orion_aitown_mcp` client (`aitown_list_players`), maps semantic kind → `{x, y}`:

- `approach_player` → position of `ref` player (or nearest non-Orion player if `ref` omitted); `denied` if none.
- `start_conversation` → resolve target (`ref` or nearest non-Orion player), approach, then send the upstream conversation-start input (`sendInput` invite/accept per `aiTown/inputs.ts`); the **speech bridge** carries the dialogue once the conversation is live. Subject to a social-initiation cooldown (`EMBODIMENT_SOCIAL_COOLDOWN_SEC`) so involuntary starts don't spam invites; `denied` if no target or cooldown active.
- `go_to_location` → lookup in a small named-location config (`EMBODIMENT_LOCATIONS_JSON`, default a handful of map spots); `denied` if unknown.
- `wander` → random reachable offset from current position within `EMBODIMENT_WANDER_RADIUS`.
- `idle` → no destination → `resolved_noop`.

Then `send_input(name="moveTo", args={playerId, destination})` → `EmbodimentOutcomeV1`.

### Idle heartbeat

When no intent has arrived for `EMBODIMENT_IDLE_HEARTBEAT_SEC` (default 0 = disabled), the worker MAY self-emit a `wander`/`idle` intent so a body with silent producers still shows minimal life. Off by default; enabling it is the "continuous presence" knob.

## Orion's body

Orion's town character is a **materialized projection of its live self-model**, not a hand-authored persona (config truth) and not regenerated every boot (non-deterministic). This keeps town-Orion coherent with the rest of the system while staying inspectable and stable.

### Source

`orion-self-state-runtime` already persists the needed artifacts:

- `IdentitySnapshotV1` — `anchor_strategy`, `summary`, `drive_pressures`, `tension_kinds`.
- `SelfStateV1` — `overall_condition`, `trajectory_condition`.

### Projection: `orion/embodiment/persona.py`

`build_orion_town_persona(identity_snapshot, self_state) -> OrionTownPersonaV1`:

| Persona field | Source | Notes |
|---------------|--------|-------|
| `name` | constant `"Orion"` | Stable identity anchor |
| `identity_blurb` | `IdentitySnapshotV1.summary` → third-person, bounded (≤ ~280 chars) | Public-safe persona line the town LLM reads |
| `plan` | derived from dominant drive / `anchor_strategy` | AI Town agents carry a "plan"; grounds behavior in current disposition |
| `spritesheet` | `EMBODIMENT_ORION_SPRITE` (config, stable) | Visual identity; a chosen upstream spritesheet |
| `provenance` | `snapshot_id` + `generated_at` of the source snapshot | Makes the persona a traceable artifact |

### Privacy boundary (hard constraint)

The town `Descriptions` surface is **frontend-visible and consumed by the town LLM** — a semi-public surface. The projection therefore:

- Emits only a **public-safe persona summary**; never raw `overall_condition`/`trajectory_condition`, journals, mirrors, or blocked material.
- Routes through the existing self-model projection/privacy boundary (same rung that already summarizes self-state for other surfaces) rather than reading raw rows directly.
- Is the reason this is a **proposal-mode-grade** change: capability = Orion gains an external persona surface; data touched = self-model summary only; privacy boundary = public-safe projection; trace = `provenance.snapshot_id`; failure mode = hollow persona; rollback = static fallback persona + flag off.

### Empty-shell guard + fallback

If the self-model is unavailable or the projected `identity_blurb` is empty (`raw_len=0`), bootstrap **refuses to write a hollow persona**. It falls back to a minimal safe persona (`name="Orion"`, neutral one-line blurb, config spritesheet) and reports `persona_source=fallback` explicitly. A hollow persona is never treated as success.

### Refresh

AI Town sets a character's description at agent creation, so the persona is stable between explicit resyncs. `bootstrap_orion_agent.py --resync-identity` rebuilds the projection from the latest snapshot and updates the live agent (via `sendInput`/re-create per upstream capability). Town-Orion tracks the self-model when you choose, not silently.

### Upstream patch

A patch under `services/orion-ai-town/patches/` adds the `"Orion"` character slot to AI Town's `Descriptions` (name + spritesheet + placeholder identity/plan), so `createAgent` can target it. Upstream stays pinned; the patch is applied by `scripts/apply_upstream_patches.sh` alongside the existing embed patch. The projected `identity_blurb`/`plan` fill the slot at bootstrap.

## Bootstrap script

`services/orion-embodiment/scripts/bootstrap_orion_agent.py` — idempotent, dry-run by default.

```text
1. Load AITOWN_* from ~/.fcc/.env (fcc env catalog).
2. If AITOWN_ORION_PLAYER_ID set AND resolves to a live player (aitown_list_players) and not --force:
     if --resync-identity: refresh the Orion description projection (see Orion's body), continue; else
     print "already embodied: player=… agent=…"; exit 0
3. Build the Orion character projection (see Orion's body) → {name, identity_blurb, plan, spritesheet}.
     If the projection is empty/unavailable: fall back to the minimal safe persona and report it.
4. Else (no live Orion body): send_input(name="createAgent", args={character:"Orion", ...projection})
     poll aitown_list_agents / aitown_list_players until the new agent/player appears (timeout).
5. Print resolved AITOWN_ORION_PLAYER_ID / AITOWN_ORION_AGENT_ID + the projection provenance (snapshot_id).
6. If --write: update ONLY those two keys in ~/.fcc/.env in place (create if absent, replace if present),
     leave all other lines/secrets byte-for-byte untouched. Print restart commands.
```

**Restart reminder printed on `--write`** (never run by the script):

```bash
# ~/.fcc/.env changed → restart consumers of AITOWN_ORION_*
docker compose --env-file services/orion-embodiment/.env -f services/orion-embodiment/docker-compose.yml up -d --build
# and, so FCC MCP telekinesis shares the same body:
docker compose --env-file services/orion-harness-governor/.env -f services/orion-harness-governor/docker-compose.yml up -d
```

## Producers

### (C) substrate dynamics tick — `orion/embodiment/drive_map.py`

Thin, pure mapping `DriveStateV1 → EmbodimentIntentV1 | None`:

| Condition (reads real pressure keys) | Intent |
|--------------------------------------|--------|
| `social` ≥ θ_social_high **and** not in a conversation **and** cooldown clear | `start_conversation` (ref omitted → nearest), `reason="social pressure {p} → initiate"` |
| dominant `social` ≥ θ_social | `approach_player` (ref omitted → nearest), `reason="social pressure {p}"` |
| dominant `predictive`/curiosity ≥ θ_curio | `wander`, `reason="predictive pressure {p}"` |
| all pressures below θ_idle | `idle` |
| otherwise | `None` (no intent this tick) |

High social pressure **escalates** approach → initiation. Involuntary `start_conversation` still obeys the resolver's social-initiation cooldown and remains preemptable by deliberate (D) intents.

Hooked into `SubstrateWorker._dynamics_tick` **after** `engine.tick()`, flag-gated by `EMBODIMENT_C_TICK_ENABLED`. Publishes with `source=involuntary`, fail-open (never raises out of the tick). Reuses the tick's existing cadence — no new timer.

> Grounding note: the tick currently logs `pressure_updates` from `SubstrateDynamicsEngine.tick()`. The C hook reads the same durable drive/pressure state the engine writes; exact pressure keys (`social`, `predictive`, …) are enumerated against the live store when wiring the producer, not invented here.

### (D) deliberate — two emit points

- **Finalize/voice pass** (`orion/harness/finalize.py`, in `orion-harness-governor`): when a unified turn produces a reply, emit a `source=deliberate` intent on the **same `correlation_id`**. Default `approach_player` toward the interlocutor when the turn is relational; `EMBODIMENT_D_FINALIZE_ENABLED`. This is the literal speech+movement-one-identity binding.
- **Cortex background/quick lane** (`orion-cortex-exec`): a background request decides movement while idle (no chat turn). Rides the `quick` lane (your "fast lane for delegated subtasks"). `EMBODIMENT_D_BACKGROUND_ENABLED`.

Both build intents through `orion/embodiment/intents.py` so the schema/`reason` contract is enforced in one place.

## Perception

Actuation without perception is an open loop. The worker already reads world state to resolve intents; it also **emits** that reading as `WorldPerceptionV1` on `orion:embodiment:perception` every `EMBODIMENT_PERCEPTION_INTERVAL_SEC` (default off = 0). Three consumers close the loop across the involuntary, identity, and deliberate paths:

### 1. Substrate loop (involuntary closure)

`orion-substrate-runtime` consumes perception and maps it to the graph via a new adapter `orion/substrate/relational/adapters/town_perception_ctx.py`, following the existing `map_self_state_ctx_to_substrate` pattern (→ `SubstrateGraphRecordV1`). Nearby players / proximity become graph nodes and drive inputs, so **C's drives move the body, and perceiving the resulting world feeds back into drives/attention.** Gated by `EMBODIMENT_PERCEPTION_SUBSTRATE_ENABLED`.

> **Loop-damping:** perception→drive→movement→perception is a live feedback loop. Perception cadence is decoupled from and slower than the drive tick, proximity contributions are bounded, and ingestion is fail-open. The loop is a slow homeostatic pull, not a tight servo — no runaway pacing.

### 2. Self-state grounding (identity closure)

`orion-self-state-runtime` consumes perception as a new best-effort observability input (same mechanism as its existing `attention_broadcast` / `hub_presence` inputs, with an age gate). Orion's `SelfStateV1` gains "I am embodied in the town, near X" grounding — the self-model knows the body exists and where it is. Gated by `EMBODIMENT_PERCEPTION_SELFSTATE_ENABLED`.

### 3. Cortex context (deliberate closure)

The latest `WorldPerceptionV1` is exposed as a compact read-model the D producers read, so deliberate movement is informed ("walk toward Juniper" resolves against a real, current view). `orion-cortex-exec` keeps the latest perception keyed by `player_id` (latest-wins, TTL'd); finalize/background producers read it when composing an intent. Gated by `EMBODIMENT_PERCEPTION_CORTEX_ENABLED`.

All three consumers are independently flag-gated and default off; perception emission itself is off until `EMBODIMENT_PERCEPTION_INTERVAL_SEC > 0`. Nothing perceives until explicitly enabled.

## Speech

AI Town's agent LLM already points at `orion-llm-gateway`, but that is the **base model**, not Orion's cognition. Same gateway ≠ same mind. To make town-Orion speak with Orion's real voice, town dialogue for Orion's agent is generated by **Orion's cortex** and injected — upstream stays unforked.

### Bridge: `orion/embodiment/speech.py`

1. Perception reports Orion is in an `active_conversation` — whether another agent engaged Orion, or Orion's own `start_conversation` intent opened it (the same `WorldPerceptionV1` the loop already emits).
2. The worker requests an utterance through the **existing cortex chat/quick lane** — the same request/reply rail Hub uses — so the reply passes through stance, memory, and self-model, not the stock town LLM. The town conversation context (interlocutor, recent lines from perception) is the prompt; the identity is Orion.
3. The worker injects the reply into the town via `send_input` conversation inputs (`startTyping` → write/`finishSendingMessage`, per upstream `aiTown/inputs.ts`).

Speech and movement therefore flow through **one worker and one identity**: the same `orion-embodiment` process, the same body `player_id`, the same Orion cognition. A town utterance is traced by the cortex turn's `correlation_id`, exactly like a Hub reply.

### Guards

- **Gated off** by `EMBODIMENT_SPEECH_ENABLED` (default false); no town speech until enabled.
- **Own-agent only:** the bridge speaks only for Orion's `player_id`; it never generates for other town agents.
- **Empty-shell guard:** an empty cortex reply is not injected (no hollow utterance).
- **Rate/turn discipline:** one in-flight utterance per conversation; the worker waits for its own injected message to land before requesting the next, bounded by the perception cadence.
- **Privacy:** the cortex turn runs under the town-conversation context; it does not surface private memory the normal chat pipeline would itself gate — the existing stance/recall privacy boundary applies unchanged.

`orion-embodiment` becomes a **client** of the existing cortex chat request/reply rail (no new speech schema; reuses the chat contracts). No existing channel's producer set changes semantically beyond adding the worker as a caller.

## Memory & continuity

Town experience is durable, not ephemeral: salient town events become biographical memory through Orion's **existing journal/recall rail** — no new memory store.

### Write path

A **salience gate** in the worker detects memory-worthy events from the material the loop already has:

- a conversation Orion completed (utterances were exchanged), or
- a first encounter with a previously-unseen agent.

It is bounded and deduped (not every tick, not every proximity). On a salient event the worker emits `JournalTriggerV1` on the existing `orion:actions:trigger:journal.v1` with the town context (who, what happened, Orion's disposition), provenance tagged as embodiment/town. The existing journaler composes the entry → `orion:journal:write` → `orion-sql-writer` → `orion:journal:index` → `orion-recall`. Gated by `EMBODIMENT_MEMORY_ENABLED` (default false).

### Read path (free continuity)

Because indexed journal entries already flow into `orion-recall`, future cortex turns recall town experience with **zero new read wiring** — both Hub chat and town speech (which runs through the same cortex pipeline) can surface "I talked with X in the town yesterday." Embodiment thereby gains continuity across sessions using the memory system Orion already has.

### Boundaries

- Reuses `JournalTriggerV1` + the journaler; `orion-embodiment` is added as a producer on `orion:actions:trigger:journal.v1` (a contract touch to record).
- Standard journal privacy applies; town memories are Orion's own biographical entries, subject to the existing recall privacy boundary.
- Salience gate prevents journal spam; a quiet town writes little.

## Failure modes, privacy, rollback

- **Convex unreachable / world down:** resolver/actuator emit `status=error` with reason; worker never crashes the loop; producers unaffected (fire-and-forget publish).
- **No live players for `approach_player`:** `status=denied`, reason logged; no move.
- **Missing `AITOWN_ORION_PLAYER_ID`:** worker refuses to actuate, emits `denied`; bootstrap is the remedy.
- **Runaway movement:** `EMBODIMENT_DELIBERATE_HOLD_SEC` + intent cadence bound frequency; idle heartbeat off by default.
- **Hollow persona:** empty self-model projection → bootstrap refuses to write, falls back to minimal safe persona, reports `persona_source=fallback` (never treated as success).
- **Perception feedback runaway:** perception cadence is slower than and decoupled from the drive tick; proximity drive contributions bounded; ingestion fail-open — a slow homeostatic pull, not a tight servo.
- **Speech:** empty cortex reply not injected; one in-flight utterance per conversation; own-agent only.
- **Social-initiation spam:** `start_conversation` (especially involuntary/C) is bounded by `EMBODIMENT_SOCIAL_COOLDOWN_SEC` and suppressed while already in a conversation; `denied` otherwise.
- **Privacy:** intents carry only movement semantics + a short `reason`; the persona projection emits a public-safe self-model summary only (never raw condition/journals/blocked material); town speech runs the normal stance/recall privacy boundary unchanged. No private material crosses the embodiment channels or into the frontend-visible `Descriptions` surface.
- **Rollback:** every path flag-gated off by default. Disable = `ORION_EMBODIMENT_ENABLED=false` (kills actuation), and the producer/perception/speech flags off. No schema migration; channels are `experimental`.
